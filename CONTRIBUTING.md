# Contributing

Thanks for contributing to ACG-Core.

## Design principles

1. Keep Core small.
2. Prefer mechanical enforcement over natural language rules.
3. Be honest about limitations.
4. Do not promote ACG-Core as complete safety.
5. Preserve compatibility with existing CI/CD workflows.

## What belongs in Core

A feature belongs in Core only if it is:

- mechanically enforceable;
- low overhead;
- useful without organizational transformation;
- compatible with common repositories.

## What belongs outside Core

Move these to ACG-Process or ACG-Full:

- semantic diff;
- behavioral baselines;
- adversarial reviewers;
- telemetry systems;
- multi-agent orchestration;
- policy engines;
- distributed ledgers.
